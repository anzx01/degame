## 产品需求文档（PRD）V2.0：《桌面整理大师》

副标题：纯 2D 桌面整理题材三消休闲游戏

| 文档版本 | 修改日期   | 修改人 | 修改内容 |
| -------- | ---------- | ------ | -------- |
| V2.0     | 2026-04-25 | Codex  | 基于原方案重构，明确 2D 方向，收敛 MVP 范围，补齐可落地玩法闭环 |

---

### 1. 产品定义

#### 1.1 一句话定义

《桌面整理大师》是一款**纯 2D、轻量级、单机休闲**的桌面整理题材三消游戏。玩家通过交换桌面杂物图标完成消除、清理障碍、恢复桌面整洁度，在 2-4 分钟的一局中获得持续的“越玩越干净”的解压体验。

#### 1.2 立项背景

“桌面很乱但整理完很爽”是天然成立的情绪场景，而三消则是最成熟的即时反馈玩法之一。原方案的方向是对的，但问题在于：

* 差异化主要停留在题材换皮，桌面整理没有真正进入关卡目标与反馈循环。
* 模式、功能和包装较多，MVP 过散，不利于快速验证产品价值。
* “解压”与“挑战”关系没有理顺，容易出现既不够轻松也不够有目标感的中间状态。
* 视觉描述偏拟真，容易把实现推向高成本方向。

因此本次重构的核心原则是：**保留桌面整理题材，强化玩法闭环，明确只做 2D，不做伪 3D。**

#### 1.3 产品定位

* 平台：Web 优先，兼容 PC 浏览器和移动浏览器
* 类型：2D 休闲三消 + 轻度关卡推进
* 核心卖点：每次消除都直接服务于“整理桌面”，不是普通三消换图标
* 用户心智：打开即玩，低学习成本，短时放松，视觉治愈

#### 1.4 本版优化结论

| 原方案问题 | 新版优化方案 |
| ---------- | ------------ |
| 主题更多体现在图标皮肤 | 增加“整洁度”“桌面恢复层”“障碍清理”“分类任务”，让题材进入核心循环 |
| 模式过多，MVP 失焦 | MVP 只做主线整理关卡，其他模式后置 |
| 想解压又想保留挑战 | 主线关卡不做硬失败，以“推荐步数/星级”承接挑战 |
| 容易向拟真或 3D 膨胀 | 明确为纯 2D 棋盘、2D 场景、2D 特效、固定镜头 |
| 参考真实 Windows 桌面 | 改为“通用桌面操作系统视觉语言”，避免直接复制真实系统资源 |

#### 1.5 产品目标

* 在 30 秒内让新用户理解玩法。
* 在 1 分钟内让用户感受到“桌面变干净”的独特爽点。
* 在 2-4 分钟内完成一局，形成“再来一局”的轻量循环。
* 用低成本 2D 内容完成题材差异化验证，而不是依赖复杂系统堆量。

#### 1.6 非目标

* 不接入真实系统文件，不读取或操作用户桌面。
* 不做 3D 桌面、镜头旋转、复杂物理模拟。
* 不在 MVP 引入强数值养成、抽卡、社交公会等重系统。
* 不在 MVP 做桌宠、PWA、截图分享、每日挑战等非核心功能。

### 2. 用户与使用场景

#### 2.1 目标用户

| 用户类型 | 典型状态 | 主要诉求 |
| -------- | -------- | -------- |
| 上班族/学生 | 短暂休息、脑力疲劳、切换注意力 | 想快速放松，但不想学复杂规则 |
| 三消用户 | 玩过经典三消，对品类规则熟悉 | 想看到新题材和更强的反馈包装 |
| 整理治愈偏好用户 | 对“从杂乱到整洁”有明显满足感 | 希望看到明显的秩序恢复与收纳成就感 |

#### 2.2 核心使用场景

* 午休、课间、通勤碎片时间，打开网页快速玩一局。
* 工作学习压力大时，用“整理杂乱桌面”的过程替代纯刷分。
* 对整理、美化、收纳题材敏感的用户，用视觉恢复过程获得治愈感。

---

### 3. 2D 产品方向定义

#### 3.1 2D 方向原则

本产品必须从立项开始就按 2D 设计，不把 2D 当作 3D 的简化版。

* **固定镜头**：始终为正视角/轻微俯视角的 2D 棋盘，不做自由旋转。
* **2D 资源生产**：所有图标、障碍、背景、按钮、特效均以 2D sprite/序列帧为主。
* **2D 恢复表现**：桌面整洁变化通过图层切换、遮罩擦除、粒子、位移动画完成。
* **2D 反馈风格**：强调清脆、利落、清扫感，而不是厚重拟真。

#### 3.2 为什么必须坚持 2D

* 更适合三消的规则可读性，用户一眼能看懂状态。
* 资源成本可控，适合快速验证题材和玩法闭环。
* Web 端性能更稳定，移动端适配难度更低。
* “整理感”并不依赖 3D，关键在于前后对比与反馈节奏。

#### 3.3 2D 表现边界

允许的表现：

* 平面图标的轻微弹性缩放
* 粒子飞散、扫除轨迹、文件吸入回收站的 2D 轨迹动画
* 背景层从脏乱到清爽的遮罩渐显
* 轻微分层视差，但不改变空间透视关系

不允许的表现：

* 3D 模型桌面
* 透视旋转棋盘
* 带真实景深的镜头推进
* 为了“高级感”额外引入高成本 3D 包装

### 4. 核心体验与设计原则

#### 4.1 核心体验

玩家不是单纯在“消方块”，而是在做一件具体且有结果感的事：**把一个混乱的桌面整理干净**。

#### 4.2 四条设计原则

* **快理解**：规则必须保留三消标准心智，降低上手门槛。
* **强反馈**：每次消除都要有“清理掉东西”的明确反馈。
* **低挫败**：允许失误，减少硬失败，强调放松。
* **可扩展**：后续内容应建立在关卡目标、障碍和主题包之上，而不是堆叠孤立系统。

#### 4.3 成功体验的判断标准

用户在首局结束后应该自然说出以下感受中的至少一个：

* “这个不像普通三消换皮，真的像在整理桌面。”
* “看着桌面慢慢干净起来很舒服。”
* “一局不长，顺手就能再来一局。”

### 5. 核心玩法闭环

#### 5.1 单局循环

1. 进入关卡，看到一张被杂物覆盖的 2D 桌面。
2. 查看本关整理任务，例如“清理 18 个截图”“移除 6 个便签”。
3. 交换相邻图标，完成三消。
4. 触发掉落、连锁和特殊整理工具。
5. 清除障碍并提升“整洁度”。
6. 桌面背景逐步从杂乱变整洁。
7. 达成目标后进入结算，展示整理前后对比。
8. 获得星级和章节进度，进入下一关。

#### 5.2 长线循环

1. 完成关卡。
2. 解锁新的桌面主题章节。
3. 看到更丰富的障碍、任务和视觉包。
4. 持续收集“已整理桌面”图鉴。

### 6. 核心玩法设计

#### 6.1 棋盘基础规则

| 项目 | 方案 |
| ---- | ---- |
| 棋盘尺寸 | MVP 固定为 8x8 |
| 基础操作 | 拖动交换，或点击两次交换相邻格 |
| 交换结果 | 只有能形成消除的交换才生效，否则回弹 |
| 消除规则 | 横向或纵向 3 个及以上相同图标连线即消除 |
| 掉落规则 | 自上而下补位，补位后继续检测连锁 |
| 无解处理 | 自动轻量洗牌，不惩罚玩家 |
| 初始生成 | 开局不出现自动消除，并保证至少有一步可走 |

#### 6.2 基础图标类型

MVP 基础图标控制在 6 类，既保证辨识度，也避免早期关卡过于复杂。

| 图标类型 | 视觉设定 | 情绪联想 |
| -------- | -------- | -------- |
| 文档 | 白纸/蓝边文档 | 工作待办 |
| 截图 | 图片缩略图 | 桌面堆积 |
| 文件夹 | 彩色文件夹 | 归类整理 |
| 压缩包 | 拉链文件包 | 临时堆放 |
| 快捷方式 | 带小箭头的图标 | 冗余入口 |
| 临时文件 | 半透明缓存块 | 杂乱来源 |

说明：

* 图标采用原创 2D 视觉，不直接复刻任何真实系统图标。
* 每类图标都需要同时具备颜色差异和轮廓差异，便于快速识别与无障碍适配。

#### 6.3 整理任务设计

新版 PRD 的关键优化是：**桌面整理不只是一句文案，而是关卡目标本身。**

MVP 关卡任务以以下三类为主：

| 任务类型 | 示例 | MVP 优先级 |
| -------- | ---- | ---------- |
| 清理指定文件 | 清理 18 个截图、12 个压缩包 | P0 |
| 移除障碍 | 去掉 6 张便签、4 个弹窗 | P0 |
| 恢复整洁度 | 将整洁度提升到 100% | P0 |

不进入 MVP 的任务类型：

* 限时冲刺
* 高分竞赛
* 连击限定任务
* 多阶段 Boss 任务

#### 6.4 整洁度系统

整洁度是区别于普通三消换皮的核心系统。

规则定义：

* 每次有效消除都会提升整洁度。
* 清除障碍会额外提升整洁度。
* 关卡背景存在“脏乱层”和“清爽层”两套 2D 图层。
* 随着整洁度提升，脏乱层逐渐淡出，清爽层逐渐显现。
* 关卡结束时，必须让用户明显看到“整理前/整理后”的对比。

用户价值：

* 给消除行为一个更具体的结果。
* 让题材差异转化为可感知的视觉目标。
* 让没有强数值系统的产品依然具备持续动力。

#### 6.5 特殊整理工具

| 触发方式 | 工具名称 | 效果 |
| -------- | -------- | ---- |
| 4 连直线 | 批量归档 | 清除一整行或一整列 |
| L/T 形消除 | 磁盘清扫 | 清除中心周围 3x3 区域 |
| 5 连直线 | 全盘搜索 | 清除某一类型全部图标 |

命名原则：

* 继续保留“桌面整理”语义，不用标准三消里的普通炸弹命名。
* 动画上强调扫除、吸附、归档、检索等动作，而不是纯爆炸。

#### 6.6 障碍系统

MVP 只保留 3 种障碍，保证内容可控。

| 障碍名称 | 视觉表现 | 规则 | MVP |
| -------- | -------- | ---- | --- |
| 灰尘层 | 覆盖格子的灰尘/污渍 | 邻近消除一次即可清除 | 是 |
| 便签 | 贴在图标上的便签纸 | 需在该格完成一次消除 | 是 |
| 弹窗 | 占据棋盘单格的小弹窗 | 邻近消除两次后关闭 | 是 |

后续可扩展但不进入 MVP：

* 锁定文件
* 病毒污染块
* 自动生成垃圾的下载器
* 多格占位的大型弹窗

#### 6.7 压力与失败设计

原方案强调解压，因此新版不建议采用重失败制。

主线关卡规则：

* 关卡有“推荐步数”，用于星级评级。
* 超过推荐步数不会直接失败，但星级会下降。
* 用户始终可以完成当前关卡，降低挫败感。
* 若连续无有效思路，系统提供高亮提示。

这样既保留关卡设计价值，也不会损害产品的放松定位。

### 7. 关卡与内容规划

#### 7.1 章节结构

MVP 建议做 4 个桌面主题章节，共 60 关，每章 15 关。

| 章节 | 主题 | 主要内容 | 新机制引入 |
| ---- | ---- | -------- | ---------- |
| Chapter 1 | 办公桌面 | 文档、截图、基础灰尘 | 基础三消、整洁度 |
| Chapter 2 | 学生书桌 | 资料包、临时文件、便签 | 便签障碍 |
| Chapter 3 | 创作者工作台 | 素材文件、压缩包、弹窗 | 弹窗障碍、组合工具 |
| Chapter 4 | 深夜电竞桌面 | 高频掉落、混合障碍 | 混合目标、难度峰值 |

#### 7.2 关卡节奏

| 阶段 | 目标 |
| ---- | ---- |
| 1-10 关 | 建立基础规则和整洁度心智 |
| 11-25 关 | 引入便签和指定文件清理 |
| 26-45 关 | 引入弹窗和多目标组合 |
| 46-60 关 | 混合障碍，测试道具理解与节奏控制 |

#### 7.3 单关时长目标

* 新手关卡：60-90 秒
* 常规关卡：2-3 分钟
* 难点关卡：不超过 4 分钟

#### 7.4 内容生产方式

* MVP 以**手工关卡模板 + 轻量参数调优**为主。
* 不建议首版完全依赖随机生成。
* 每章应确保“任务目标、障碍、背景恢复效果”三者一致，而不是只换背景图。

### 8. 轻度成长与长期目标

#### 8.1 成长设计原则

本产品不做重数值，但需要轻量成长来支撑留存。

#### 8.2 MVP 成长内容

| 系统 | 方案 |
| ---- | ---- |
| 星级系统 | 每关根据推荐步数完成情况给 1-3 星 |
| 章节解锁 | 累计星级解锁后续章节 |
| 桌面图鉴 | 完成关卡后解锁“整理完成”的桌面图卡 |
| 最佳记录 | 保存单关最佳步数和最高连锁 |

#### 8.3 后续可扩展方向

* 无尽模式
* 每日挑战
* 成就系统
* 自定义壁纸奖励

这些都不应阻塞 MVP。

### 9. 界面与交互设计

#### 9.1 主界面

主界面应简洁，不做复杂大厅。

核心入口：

* 开始整理
* 继续上次进度
* 章节选择
* 设置

#### 9.2 关卡内 HUD

| 区域 | 内容 |
| ---- | ---- |
| 顶部 | 当前关卡、整洁度、暂停按钮 |
| 左侧/上方 | 本关任务列表 |
| 中央 | 8x8 2D 棋盘 |
| 底部 | 推荐步数、当前步数、提示按钮 |

PC 和移动端布局规则：

* PC 端以横向信息布局为主，保证棋盘居中。
* 移动端以纵向堆叠为主，任务信息收纳在顶部。
* 所有关键交互区域必须单手可触达。

#### 9.3 结算页

结算页需要强化“整理完成”的满足感。

必备内容：

* 整理前/整理后的小型对比
* 本关星级
* 本局连锁峰值
* 下一关按钮

不建议在首版结算页塞入太多二次入口。

#### 9.4 新手引导

新手引导应只覆盖前 3 关。

| 关卡 | 引导内容 |
| ---- | -------- |
| 第 1 关 | 教玩家如何交换和形成三消 |
| 第 2 关 | 教玩家理解指定文件清理 |
| 第 3 关 | 教玩家理解整洁度与障碍清除 |

要求：

* 每次引导只讲一件事。
* 引导尽量用演示和高亮，不用大段文字。
* 允许跳过。

### 10. 视觉与音频方向

#### 10.1 美术关键词

* 纯 2D
* 清爽
* 轻拟物
* 卡通桌面感
* 有秩序恢复的满足感

#### 10.2 美术风格定义

* 棋盘元素采用统一的原创图标系统，边角圆润，轮廓清晰。
* 桌面背景采用章节化主题设计，每章有明显的“脏乱版”和“整理完成版”。
* 特效不强调大爆炸，强调吸走、归档、擦除、折叠、关闭、扫开。
* 用色避免过于接近真实 Windows 资源，建立自己的视觉品牌。

#### 10.3 动效原则

* 图标交换：轻微位移 + 回弹
* 消除反馈：缩放、碎屑、吸附轨迹
* 道具触发：线性清扫、范围吸附、全盘检索波纹
* 整洁度提升：背景污渍被擦去、桌面亮度和饱和度轻微提升

#### 10.4 音效方向

* 基础点击音：清脆、短促
* 消除音：纸张、文件夹、压缩、吸附感
* 连锁音：逐步升调，但不过度刺激
* 结算音：轻快、收束感强

要求：

* 默认音量温和，不制造噪音压力。
* 必须提供 BGM、SFX 独立开关。
* 移动端震动反馈默认关闭，需用户主动开启。

### 12. 非功能需求

#### 12.1 性能

| 项目 | 目标 |
| ---- | ---- |
| 帧率 | 常规设备稳定 60fps，低端设备不低于 30fps |
| 首屏加载 | 首次可玩时间不超过 5 秒 |
| 资源控制 | 首包尽量控制在 10MB 以内，按需加载章节资源 |
| 特效预算 | 粒子数量可按设备档位降级 |

#### 12.2 兼容性

* PC：Chrome、Edge 最新主流版本优先。
* 移动：iOS Safari、Android Chrome 主流版本优先。
* 最低适配分辨率：
  * PC：1366x768
  * Mobile：390x844

#### 12.3 可访问性

* 图标必须同时具备颜色和形状区分。
* 提供高对比度模式。
* 重要目标不只靠颜色提示。

#### 12.4 存档与数据

* 默认使用浏览器本地存储保存进度。
* 在线数据统计独立于本地存档，不影响单机可玩性。
* 首版不强依赖登录系统。

#### 12.5 技术边界

* 渲染层明确为 2D。
* 不使用 3D 资源作为主要内容生产方式。
* 所有后续功能必须服从“2D、轻量、快反馈”的方向，不可反向拉高复杂度。

### 13. 数据指标（KPI）

MVP 验证阶段建议重点看以下指标：

| 指标 | 目标 |
| ---- | ---- |
| 新手引导完成率 | >= 85% |
| 首日留存（D1） | >= 35% |
| 平均单次游戏时长 | >= 6 分钟 |
| 首次会话完成 5 关比例 | >= 60% |
| 首次会话完成 10 关比例 | >= 40% |
| 音效开启率 | >= 50% |
| 主观“解压”正向反馈率 | >= 80% |

补充说明：

* 由于产品以情绪价值为核心，主观反馈指标与留存同样重要。
* 若用户普遍只觉得“像普通三消”，说明题材差异化没有打透。

### 15. 风险与应对

| 风险 | 表现 | 应对措施 |
| ---- | ---- | -------- |
| 题材差异化不足 | 用户觉得只是图标换皮 | 用整洁度、任务、障碍、前后对比强化主题闭环 |
| 解压感不足 | 用户觉得被步数限制和失败惩罚打断 | 不做硬失败，以推荐步数承接挑战 |
| 视觉侵权风险 | 过度接近真实 Windows 视觉 | 坚持原创图标体系和通用桌面语言 |
| 性能波动 | 多设备上掉帧或特效卡顿 | 2D 资源图集化、分档粒子、按需加载 |
| 内容消耗快 | 关卡打完后无继续动力 | 提前预留无尽模式、图鉴、每日挑战作为后续版本内容 |

---

### 16. 最终立项建议

如果只保留一句话作为执行原则，应是：

**这不是“桌面皮肤版三消”，而是一款围绕“把混乱桌面整理干净”构建完整反馈闭环的纯 2D 休闲游戏。**

因此，开发和设计阶段所有取舍都应围绕以下判断：

* 这个功能是否强化了“整理感”？
* 这个实现是否能在 2D 中高质量完成？
* 这个需求是否属于 MVP 必须验证的核心价值？

只要这三个问题的答案不够明确，就不进入首版。
